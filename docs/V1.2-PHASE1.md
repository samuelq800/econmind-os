# EconMind OS V1.2 — Phase 1

## Scope

- Unified model registry in `lib/models/registry.ts`
- Authenticated Economist Workspace at `/dashboard`
- Economic Sandbox at `/sandbox`
- Price Controls at `/models/price-controls`
- Reusable saved-run loading, duplication, deletion, favorites, and explicit completion
- Rule-based recommendations in `lib/recommendations/`

## Low-usage Supabase design

The existing `model_runs` table remains the only saved-scenario table. The Phase 1 migration adds a `metadata jsonb` column and a compact `recent_activity` table.

`recent_activity` has one unique row per user, module, and activity type. Repeated events increment `event_count` instead of appending unlimited rows. `learning_progress` remains one row per user and module.

Database writes occur only when:

- a signed-in user first opens a module in a browser tab;
- the user explicitly clicks **Run Simulation**;
- the user explicitly saves a scenario;
- the user changes a favorite;
- the user explicitly clicks **Mark complete**.

Slider and number-input changes never contact Supabase.

## Migration

After the initial schema, run:

`supabase/migrations/20260722010000_v1_2_phase1.sql`

It adds indexes, an update trigger, RLS policies, authenticated RPC functions, and a model-run trigger that updates progress without an extra frontend request.

## Sandbox assumptions

The Sandbox uses standardized deterministic coefficients. Fiscal, monetary, market-regulation, environmental, and trade contributions are calculated separately and then summed. Every input is sanitized to its declared range and every output is clamped.

The result communicates directional relationships and trade-offs. It is not calibrated to a country, dataset, historical episode, or forecasting model.
