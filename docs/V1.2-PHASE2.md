# EconMind OS V1.2 — Phase 2

Phase 2 expands the browser-side model library without adding a database migration or any background Supabase writes.

## New interactive models

- Externalities: private versus social cost, efficient quantity, corrective tax or subsidy, and welfare recovery.
- Monopoly: MR = MC output, monopoly price, profit, markup, Lerner index, and deadweight loss.
- Production Possibility Frontier: allocation, capacity use, rising opportunity cost, growth shifts, and point classification.
- Aggregate Demand & Supply: demand and supply shocks, short-run equilibrium, output gap, price pressure, and a simplified unemployment-gap relationship.

Every slider calculation is deterministic TypeScript running in the browser. Results are educational illustrations, not forecasts.

## Discovery and progression

- The model registry marks all four models as available.
- The model library supports text, category, difficulty, and availability filters.
- The sidebar, homepage, dashboard progress, cloud library, favorites, and learning records continue to derive from the central registry.
- Recommendations now point only to available models.

## Data usage

- Parameter state and A/B snapshots stay in localStorage.
- Supabase is contacted only for explicit authentication, favorites, completion, and named-run actions, plus one aggregated visit record per signed-in browser tab.
- Existing `model_runs.parameters` and `model_runs.results` JSONB columns accept the new models without schema changes.
- No service role key, paid server, AI API, or scheduled background service is used.
